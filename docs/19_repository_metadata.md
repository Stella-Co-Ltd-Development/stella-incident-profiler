# Repository Metadata

## Canonical repository name

`stella-incident-profiler`

## Repository description

Open-source JavaFX desktop and MCP-enabled incident profiler for AWS-hosted Java web applications using JFR, OpenTelemetry, CloudWatch, ALB logs, and Aurora metrics.

## README positioning

The README must describe the project as an open-source incident investigation platform, not as an internal-only tool.

The first public README should include:

- problem statement
- architecture summary
- screenshots or mock-mode screenshots
- debug/mock mode quick start
- contribution guide link
- security policy link
- license status
- maintainer information

## Name rationale

`stella-incident-profiler` is short, vendor-neutral enough for OSS discovery, and broad enough to cover future support beyond Java profiling, including AWS incident timelines, MCP tools, and observability correlation.
